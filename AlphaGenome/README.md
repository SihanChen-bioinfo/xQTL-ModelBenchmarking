# AlphaGenome for Functional xQTL Prediction

This directory contains scripts for functional xQTL prediction using variant effect features generated by AlphaGenome. The workflow first predicts regulatory effects of variants using AlphaGenome, and then trains machine learning models to classify functional xQTL variants.

---

## Overview

The workflow consists of two steps:

1. Predict variant regulatory features using AlphaGenome.
2. Train and evaluate xQTL prediction models using AlphaGenome-derived features.

---

## Step 1: Generate AlphaGenome Features

AlphaGenome is a deep learning framework developed for predicting genome-wide regulatory activity and variant effects from DNA sequence.

For the original implementation, see:

https://github.com/google-deepmind/alphagenome

### Example

```bash
python runAlphaGenome.py input.vcf output_dir
```

These outputs are used as input features for downstream xQTL prediction.

---

## Step 2: Train an xQTL Prediction Model

After generating AlphaGenome features, use `AlphaGenome_predict_xQTL.py` to train and evaluate models.

### Example

```bash
python AlphaGenome_predict_xQTL.py \
    --mode train \
    --model xgb \
    --folds 5 \
    --auto_tune \
    --csv_glob "*.csv" \
    --labels_csv labels.txt \
    --metrics "delta_mean,delta_max,foldchange_mean,foldchange_max" \
    --out_dir out_dir
```

---

## Parameters

| Parameter | Description |
|------------|------------|
| `--mode` | Running mode. Choose from `train` or `predict`. Default: `train`. |
| `--csv_glob` | Glob pattern for AlphaGenome output CSV files. Each CSV must contain `id`, `feature_name`, and feature metric columns. Required. |
| `--labels_csv` | Label file containing variant IDs and binary labels. Required for training mode. |
| `--id_col` | Column name for variant identifiers in the label file. Default: `id`. |
| `--label_col` | Column name for binary labels. Default: `label`. |
| `--metrics` | Comma-separated list of feature aggregation metrics used for model input. Options include: `delta_mean`, `delta_max`, `foldchange_mean`, `foldchange_max`. Default: `delta_mean`. |
| `--folds` | Number of chromosome-based folds used for hyperparameter tuning on training data. Default: `5`. |
| `--out_dir` | Output directory for models, evaluation results, and predictions. Required. |
| `--model` | Machine learning model type. Supported options: `rf` (Random Forest), `xgb` (XGBoost), `l1` (L1 logistic regression), `enet` (Elastic Net logistic regression). |
| `--n_jobs` | Number of parallel CPU jobs (used for Random Forest). Default: `-1`. |
| `--auto_tune` | Enable automatic hyperparameter tuning using chromosome-based cross-validation on training data. |
| `--no_auto_tune` | Disable hyperparameter tuning and use default model parameters. |
| `--random_state` | Random seed for reproducibility. Default: `0`. |
| `--chrom_col` | Chromosome column name. If missing, chromosome is inferred from `id` (prefix before `:` or `_`). Default: `chrom`. |
| `--chunksize` | Chunk size for reading large CSV files. Used for memory-efficient processing. `0` disables chunking. Default: `200000`. |
| `--model_dir` | Directory containing trained models for prediction mode. Must match training model type. |
| `--predict_out` | Output filename for prediction results. Default: `predictions_{model}.csv`. |

---

## Output

The output directory contains:

- Trained models
- Performance metrics (AUROC, AUPRC, MCC, etc.)
- Feature importance scores (if supported)
- Prediction results (in predict mode)
