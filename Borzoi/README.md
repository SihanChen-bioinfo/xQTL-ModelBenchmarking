# Borzoi for Functional xQTL Prediction

This directory contains scripts for functional xQTL prediction using sequence-derived regulatory features generated by Borzoi. The workflow first predicts variant effects on chromatin and transcriptional regulatory features using Borzoi, and then trains machine learning models to identify functional xQTL variants.

## Overview

The workflow consists of two steps:

1. Predict variant effects using Borzoi (`hound_snp.py`).
2. Train and evaluate xQTL prediction models using the generated Borzoi features.

---

## Step 1: Generate Borzoi Features

For installation instructions and model resources, please refer to the original Borzoi repository:

https://github.com/calico/borzoi

Borzoi predicts the impact of genomic variants on thousands of regulatory tracks. Variant effect scores can be generated using `hound_snp.py`.

### Example

```bash
python baskerville/src/baskerville/scripts/hound_snp.py \
    -u \
    -f borzoi/hg38.fa.gz \
    -o output_dir \
    --rc \
    --stats SUM,logD2,SAX \
    -t borzoi/examples/targets_human.txt \
    borzoi/examples/params_pred.json \
    borzoi/examples/saved_models/f3c0/train/model0_best.h5 \
    input.vcf
```

---

## Step 2: Train an xQTL Prediction Model

After generating Borzoi predictions, use `borzoi_predict_xQTL.py` to train and evaluate machine learning models.

### Example

```bash
python borzoi_predict_xQTL.py \
    --mode train \
    --model xgb \
    --auto_tune \
    --folds 5 \
    --borzoi_glob "*h5" \
    --h5_datasets "logD2,SUM,SAX" \
    --labels_csv label.txt \
    --out_dir out_dir
```

---

## Parameters

| Parameter | Description |
|------------|------------|
| `--mode` | Running mode. Choose from `train` or `predict`. Default: `train`. |
| `--borzoi_glob` | Glob pattern specifying Borzoi output files (CSV, TSV, or HDF5). Multiple files will be concatenated. Required. |
| `--labels_csv` | Label file containing variant identifiers and binary labels. Required for training mode. |
| `--id_col` | Column name containing variant identifiers in the label file. Default: `id`. |
| `--label_col` | Column name containing binary labels. Default: `label`. |
| `--h5_key` | Optional pandas HDF5 key used to read a specific table from an HDF5 file. |
| `--h5_dataset` | Single HDF5 dataset to use as features (e.g., `logD2`, `SUM`, or `SAX`). |
| `--h5_datasets` | Comma-separated list of HDF5 datasets to concatenate as features. Overrides `--h5_dataset`. Example: `logD2,SUM,SAX`. |
| `--h5_use_target_labels` | Use `target_labels` stored in the HDF5 file as feature names when available. Enabled by default. |
| `--id_delim` | Delimiter used when constructing variant IDs from chromosome, position, reference allele, and alternate allele. Choices: `:` or `_`. Default: `_`. |
| `--folds` | Number of chromosome-based folds used for hyperparameter tuning on the training chromosomes. Default: `5`. |
| `--out_dir` | Output directory for trained models, evaluation metrics, and prediction results. Required. |
| `--model` | Machine learning model to use. Supported options: `rf` (Random Forest), `xgb` (XGBoost), `l1` (L1-regularized Logistic Regression), and `enet` (Elastic Net Logistic Regression). |
| `--n_jobs` | Number of CPU cores used for parallel processing. Use `-1` to utilize all available cores. Default: `-1`. |
| `--auto_tune` | Enable automatic hyperparameter tuning using chromosome-based cross-validation on the training set (default behavior). |
| `--no_auto_tune` | Disable automatic hyperparameter tuning and use built-in default hyperparameters. |
| `--random_state` | Random seed used for model training and chromosome splitting. Default: `0`. |
| `--chrom_col` | Chromosome column name used for chromosome-based cross-validation. Default: `chrom`. |
| `--model_dir` | Directory containing previously trained models for prediction mode. If not specified, `--out_dir` is used. |
| `--predict_out` | Name of prediction output CSV file. Default: `predictions_{model}.csv`. |
