# ExPecto for Functional xQTL Prediction

This directory contains scripts for functional xQTL prediction using chromatin feature differences generated by ExPecto. The workflow first computes variant-induced chromatin feature changes using ExPecto, and then trains machine learning models to predict functional xQTL variants.

---

## Overview

The workflow consists of two steps:

1. Predict chromatin feature changes using ExPecto.
2. Train and evaluate xQTL prediction models using ExPecto-derived features.

---

## Step 1: Generate ExPecto Chromatin Features

This step uses ExPecto to compute chromatin feature differences for variants in a VCF file.

For original implementation and documentation, please refer to:

https://github.com/FunctionLab/ExPecto

### Example

```bash
python runExPecto.py input.vcf
```

### Output

ExPecto generates `.diff.h5` files that store chromatin feature differences across multiple shifts and genomic contexts.

---

## Step 2: Train an xQTL Prediction Model

After generating chromatin feature files, use `ExPecto_xQTL_CV10.py` to train and evaluate models.

### Example

```bash
python ExPecto_xQTL_CV10.py \
    --mode train \
    --model l1 \
    --auto_tune \
    --folds 5 \
    --vcf input.vcf \
    --h5_glob input_features.shift_0.diff.h5 \
    --labels_csv labels.txt \
    --feature_meta deepsea_beluga_2002_features.tsv \
    --out_dir out_dir
```

---

## Parameters

| Parameter | Description |
|------------|------------|
| `--mode` | Running mode. Choose from `train` or `predict`. Default: `train`. |
| `--vcf` | Input VCF file used for ExPecto feature generation. Must match the order used in `runExPecto.py`. Required. |
| `--h5_glob` | Glob pattern for ExPecto `.diff.h5` output files (e.g., `input_features.shift_*.diff.h5`). Required. |
| `--labels_csv` | Label file containing variant IDs and binary labels. Required for training mode. |
| `--id_col` | Column name for variant identifiers in the label file. Default: `id`. |
| `--label_col` | Column name for binary labels. Default: `label`. |
| `--feature_meta` | Optional metadata file describing chromatin features (TSV/CSV). Must include columns: `number`, `Cell type`, `Assay`, `Treatment`, `Assay type`, `Source`, `link`. Feature names are constructed as `Cell|type_Assay|Treatment` following the order of `number`. |
| `--dist_strand_csv` | Optional file with columns `id`, `dist`, `strand (+/-)`. Used to build ExPecto positional kernels. If not provided, aggregation across shifts is used instead. |
| `--agg` | Aggregation strategy across shifts. Options: `mean`, `max`, or `concat`. Default: `mean`. |
| `--folds` | Number of chromosome-based folds used for hyperparameter tuning on the training set. Default: `5`. |
| `--out_dir` | Output directory for model results and predictions. Required. |
| `--model` | Machine learning model type. Options: `xgb`, `rf`, `l1` (L1 logistic regression), `enet` (elastic net logistic regression). |
| `--random_state` | Random seed for reproducibility. Default: `0`. |
| `--auto_tune` | Enable automatic hyperparameter tuning using chromosome-based cross-validation on training data. |
| `--no_auto_tune` | Disable hyperparameter tuning and use default model parameters. |
| `--model_dir` | Directory containing trained models for prediction mode. Must match training model type. |
| `--predict_out` | Output filename for prediction results. Default: `predictions_{model}.csv`. |

## Output

The output directory contains:

- Trained models
- Performance metrics (AUROC, AUPRC, MCC, etc.)
- Feature importance (if supported)
- Prediction results (in predict mode)
